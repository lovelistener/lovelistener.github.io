maven的简单了解：

  • 是 Apache 下的一个纯 Java 开发的开源项目
  • 是一个项目管理工具，可以对 Java 项目进行构建、依赖管理。
  • POM（项目对象模型，项目对象模型）是Maven工程的基本工作单元，是一个XML文件，包含了项目的基本信息，用于描述项目如何构建，声明项目依赖，等等。
  • 执行任务或目标时，Maven会在当前目录中查找POM。它读取POM，获取所需的配置信息，然后执行目标。
  • <！-模型版本-> < modelVersion > 4.0.0 </ modelVersion >
  • <！-公司或组织的唯一标志，并且配置时生成的路径也是转换生成，如com.companyname.project-group，maven会将该项目打成的jar包放本地路径：/ com /公司名称/项目组-> **< groupId >**com.companyname.project-group </ groupId >
  • <！-项目的唯一ID，一个groupId下面可能多个项目，就是靠artifactId来区分的-> < artifactId >
  • <！-版本号-> 1.0
  • 所有POM文件都需要项目元素和三个必需长度：groupId，artifactId，version。
  • 所有的POM都继承自一个父POM（无论是否显式定义了这个父POM）。父POM包含了一些可以被继承的样式设置。因此，当Maven发现需要下载POM中的依赖时，它会到Super POM中配置的替代仓库http://repo1.maven.org/maven2去下载。

AOP细节一：容器保存的是代理对象

  • 一个service类继承了接口，用**@service注册，然后被切面类切入了方法**的话，getBean(接口.class)得到的bean是代理类
    • 如果用getBean(实现类.class)得不到bean，因为service类注册好后实际上注册的是它的代理类的对象，这里的代理类是class com.sun.proxy.$Proxy12，代理类和目标类（service类）不是同一个类，但是有同一个接口
    • 上述过程的前提是有切面，
    • 也可以getBean(类名首字母小写即id)，但是也必须用一个接口来接收
    • 如果service类没有实现接口，那么不会使用jdk规定的类com.sun.proxy.$Proxy12，而是使用cglib自动创建的代理对象

AOP细节二：切入点表达式

  • *：
    • 匹配一个或多个字符 .*(int,int)
    • 匹配任意一个参数 .add(*,int)
    • 匹配任意一层路径 com.atguigu.*.MyXXX
  • ..：
    • 匹配任意参数 .add(..)
    • 匹配任意多层路径 com..Myxxx
  • &&：
    • 同时满足两个execution
  • ||：
    • 满足任意一个execution
  • !：
    • 满足不是此execution

AOP细节三：通知方法的执行顺序

  • 正常执行：before==after==afterReturning
  • 异常执行：before==after==afterThrowing

AOP细节四：在通知方法执行的时候，拿到目标方法的详细信息

  • 使用JoinPoint,因为JoinPoint对象封装了SpringAop中切面方法的信息
  • 通过getSignature()方法获取方法标签
  • 通过getArgs()方法获取参数列表

AOP细节五：拿异常信息和返回值

  • @AfterReturning(value=”execution()”,returning=”返回值赋给的变量名”)
  • 注意这个变量需要作为logReturn方法的一个参数
  • @AfterThrowing(value=”execution()”,throwing=”exception”)
  • 这个exception也要作为logException方法的一个参数

AOP细节六：通知方法的参数表不能乱写

  • spring在切入一个通知方法时，会通过反射确定每一个参数的值，比如：jointpoint，result等等，不能乱写

实验十八~二十二：@Autowired的使用

  • 配置文件中扫描包com.atguigu
  • 类bookservlet中private service，类service中dao，dao中一句输出
  • servlet doget方法，service save方法
  • 以上所有的private属性都没有new，如果什么都不加，会报空指针异常
  • 在private属性service和dao上加@autowired注解，表示自动为这个属性装配
  • 说明:
    • 以上程序依赖于三样东西：配置文件，@Controller/Service/Repository，@Autowired
    • 运行顺序为：@Controller/Service/Repository将当前类在配置文件中注册为一个bean，然后@Autowired将注册好的bean配置给类的属性
    • 核心步骤就是autowired注解在配置文件中找属性对应的bean，找的过程如下
      • 先按照类型找 getbean(class)
      • 如果找不到，抛异常
      • 如果找到多个，按照 id=”属性名” 来找
      • 如果找不到，报错，但是可以使用 @Qualifier(“配置文件中存在的bean的id名”) 注解加在对应的属性上，这样的话spring在自动装配时，会在配置文件中找这个id名对应的bean而不是按照属性名去查找
      • 如果 @Qualifier 都找不到，就会报错。
      • 如果自动装配的注解写为：＠Autowired(Required=false)
    • autowired注解可以用在方法形参上
      • 如果在配置文件中配置一个对象bean1，这个对象所属的类有一个autowired注解的方法func(BookDao bookDao)，那么在这个bean1在容器启动的时候会被创建，同时对应的func也会执行
    • autowired注解和resource注解之间的区别
      • resource注解是javaEE中的
      • autowired注解是spring中的，功能更强大，比如可以设置required=false
      • resource注解是java标准，功能较基础，但是可扩展性强，如果切换成spring之外的容器框架，还是可以使用，但是spring不行。

Spring的单元测试

      • spring于是给bookservice装配好了bookdao，user同理

    • 如果遇到多个具有相同数量的参数的构造器重载的情况，那么这些构造器的参数的取同一个index对应的参数不同，需要通过在constructor-arg下建立子标签type来进行定位。
    • 一般情况下以第一种方法通过name为属性赋值为主

实验五（重要）

  • 说明：

    • 工厂模式创建对象
    • 两种工厂：
      • 静态工厂：直接调用 工厂类.工厂方法 获得对象
      • 实例工厂：新建工厂类对象 调用工厂方法 获得对象
    • 两种工厂的区别：
      • 静态工厂 类内工厂方法加static
      • 实例工厂 使用时需要先new

  • 方法（静态工厂）：

    • bean id class=”工厂类而非产品类” factory-mothod=”生产方法” { { constructor-arg name value
    • 其中的factory-method的作用是提醒这里要创建产品对象而不是工厂对象，并且指明了生产方法
    • 这里的constructor-arg表面是给构造器传参实际上是给生产方法传参

  • 方法（实例工厂）：

    • 先配置工厂对象
      • bean id class
    • 再配置产品对象
      • bean id class=”产品类” factory-bean=”工厂bean” factory-method=”生产方法” { { constructor-arg name value
    • 说明：这种方法创建的产品不是反射创建的，而是工厂new出来的

  • 通过 FactoryBean 接口创建类来实现工厂

    • 实现三个方法：getObject，getObjectType，isSingleton
      • getObject：是工厂的生产方法
      • getObjectType：返回Xxx.class
      • isSingleton：是单例吗？返回True或false
    • 在xml文件中注册：
      • bean id=”id01” class=”实现接口的工厂的类路径”
    • 测试：
      • ioc.getBean(“id01”) 会返回产品对象，这个对象不是在加载xml文件时创建的，而是在用getBeab 方法获取的时候创建的

实验十三：基于xml的自动配置

  • 如果自定义了类A，它有许多属性，如果想在xml文件中为此类的一个对象a进行配置，则需要对这些属性依次配置
  • 现在发现xml中有的bean正好对应部分属性，并且想让xml自动把这些bean配置给这些属性，就可以使用自动配置
  • 使用方法为使用 bean autowire=””
    • default：不自动装配
    • byName：通过属性名=bean的id名 来进行配置
    • byType：通过属性类型=bean的类型 来进行配置
    • constructor：通过构造器来进行装配
      • 先按照构造器参数的类型在xml文件中找bean
      • 如果找不到，配置null
      • 如果找到多个，按照参数名称找bean
      • 如果找不到，配置null
      • 不管找到找不到，都不会报错
